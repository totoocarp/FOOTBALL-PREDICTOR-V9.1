Analiza exhaustivamente el archivo "fallosenelmodeloactual.txt" incluido dentro del proyecto.

NO implementes una única solución por problema.

Para cada fallo detectado debes:

1. Identificar la causa exacta o las posibles causas.
2. Enumerar TODAS las soluciones viables.
3. Comparar ventajas y desventajas de cada solución.
4. Determinar cuál es la más robusta para un proyecto escalable.
5. Detectar si existen problemas relacionados que aún no fueron reportados.
6. Proponer pruebas de validación para confirmar que el problema quedó resuelto.
7. Implementar la solución elegida.
8. Verificar que la implementación realmente modifica el comportamiento esperado.
9. Agregar logs de diagnóstico cuando corresponda.
10. Corregir tanto backend como frontend si el problema involucra ambos.

IMPORTANTE:

No asumir que el error está en una única función.

Realizar trazabilidad completa del flujo:

Frontend → API → Servicios → Base de Datos → Simulación → Renderizado → Logs.

Buscar especialmente errores silenciosos donde la aplicación devuelve HTTP 200 pero no realiza ninguna acción real.

SECCIÓN 1 - PREDICTOR
────────────────────────────────────

Problema detectado:

Las predicciones parecen prácticamente idénticas entre partidos completamente distintos.

Ejemplos observados:

* Brighton vs Manchester United
* Manchester United vs Brighton
* Manchester City vs Deportivo Riestra

Todos generan resultados muy similares:

* Probabilidades casi iguales
* xG casi iguales
* Distribuciones de marcadores casi idénticas
* Confianza idéntica
* Predicciones por variable muy parecidas

Investigar exhaustivamente:

Posibles causas:

* Variables no se están cargando.
* Variables siempre retornan valores por defecto.
* APIs fallan y activan fallback constante.
* Cache devuelve siempre los mismos datos.
* Escalado incorrecto.
* Pesos nunca se aplican.
* Monte Carlo usa lambdas fijos.
* Monte Carlo ignora datos reales.
* Error en normalización.
* Error en agregación de variables.
* Error en cálculo de ratings.
* Error en cálculo de xG.
* Error en construcción de features.
* Error en consulta de base de datos.
* Error en lectura de equipos.
* Error en selección de competición.
* Error en asignación local/visitante.
* Error en predictor por variable.
* Error en predictor completo.
* Error en serialización.
* Error en frontend que muestra datos incorrectos.

Validar que:

Manchester City vs Deportivo Riestra produzca una predicción radicalmente distinta a:

Brighton vs Manchester United.

Agregar herramientas de diagnóstico que permitan visualizar:

* Variables originales.
* Variables normalizadas.
* Pesos aplicados.
* Rating final.
* Lambda final.
* Probabilidades finales.
